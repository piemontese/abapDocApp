Interface IF_SERIALIZABLE_OBJECT Serializable Object

Main Package
Documentation

Meaning

Durch Implementieren dieses Tag-Interfaces wird eine Klasse (und ihre Unterklassen) als serialisierbar gekennzeichnet. Das bedeutet, dass Instanzen dieser Klasse mit dem ABAP Befehl CALL TRANSFORMATION nach XML oder JSON serialisiert und von XML oder JSON deserialisiert werden können.

Das Implementieren dieses Interfaces erzwingt keine Methoden- oder Attributdefinitionen.

Klassendesigner sollten sich bewußt sein, daß bei der Default-Serialisierung von Objekten auch private Attribute in das XML- oder JSON-Dokument abgelegt werden, sie die Serialisierung also gegebenenfalls abändern sollten.

Bei den meisten Klassen ist es sinnvoll, diese als serialisierbar zu kennzeichnen.

Das Interface darf nur in einer Klasse eines Pfads des Vererbungsbaums implementiert werden und wirkt ab dieser Klasse für alle Unterklassen.

Use

Bei der Default-Serialisierung werden alle Instanz-Attribute serialisiert, die in serialisierbaren Klassen definiert sind; Interface-Attribute werden dann serialisiert, wenn die Klasse serialisierbar ist, die das entsprechende Interface das erste Mal (bezüglich der Klassenhierarchie) implementiert.

Für Instanzen einer Klasse kann die Serialisierung derjenigen Attribute verändert werden, die in der Klasse definiert sind. Dazu werden zwei Methoden SERIALIZE_HELPER und DESERIALIZE_HELPER definiert, deren Parameter anstelle der Attribute bei der Serialisierung bzw. Deserialisierung verwendet werden.

Die Attribute einer Klasse werden mit einer Versionsnummer serialisiert, die durch die optionale Klassenkonstante SERIALIZABLE_CLASS_VERSION bestimmt wird; wenn definiert, muss diese Konstante vom Typ I sein. Beim Deserialisieren wird überprüft, ob die serialisierte Versionsnummer mit der aktuellen übereinstimmt, andernfalls wird eine Ausnahme der Klasse CX_XSLT_DESERIALIZATION_ERROR ausgelöst.

Folgende Bedingungen müssen erfüllt sein, sie werden vom Syntax-Check überprüft:

  • Die Methoden SERIALIZE_HELPER und DESERIALIZE_HELPER müssen beide definiert oder beide nicht definiert sein. Wenn beide nicht definiert sind, wird die Default-Serialisierung verwendet.
  • Beide Methoden müssen private Instanzmethoden sein.
  • Die Methode SERIALIZE_HELPER darf nur EXPORTING-Parameter haben, die Methode DESERIALIZE_HELPER nur IMPORTING-Parameter.
  • Für jeden Parameter in SERIALIZE_HELPER muss es einen Parameter gleichen Namens in DESERIALIZE_HELPER geben. Zusätzliche Parameter von DESERIALIZE_HELPER müssen optional sein.
  • Die Methode SERIALIZE_HELPER darf keinen Parameter SERIALIZABLE_CLASS_VERSION haben. Die Methode DESERIALIZE_HELPER kann einen solchen Parameter haben, der vom Typ I sein muss, wenn definiert.

Hat die Methode DESERIALIZE_HELPER einen optionalen Parameter SERIALIZABLE_CLASS_VERSION, wird der oben erwähnte Versionstest nicht vorgenommen.
